# ICE-SRC-012 - Executive Summary Writing Guidance

## Source role

Contextual editorial guidance for producing an executive-facing, decision-oriented summary.

## Adopted principles

- Treat the Executive Summary as the publication's strategic pitch, not as a passive recap.
- Lead with the problem and state the conclusion early.
- Structure the argument around problem, solution, proof, and ask.
- Optimize the first page for a sixty-second scan.
- Use bold anchor phrases and disciplined visual hierarchy.
- Exclude component-level technical detail from the first page.
- Replace vague benefits with verified evidence, clearly labeled targets, or qualified value hypotheses.
- End with a specific strategic next action.

## Project ICE adaptation

Project ICE uses a two-layer structure: a 300-500-word Executive Decision Brief followed by an 800-1,200-word supporting Executive Summary. This preserves executive scannability while providing enough strategic and architectural credibility for a flagship SAP technical publication.

## Limitations

The source's example percentages, financial figures, timelines, and budget requests are generic placeholders and are not HANA-X claims.
